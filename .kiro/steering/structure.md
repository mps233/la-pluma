# 项目结构

## 目录组织
建议的项目结构：

```
.
├── src/                    # 源代码目录
│   ├── frontend/          # 前端代码
│   │   ├── components/    # UI 组件
│   │   ├── pages/         # 页面
│   │   ├── services/      # API 调用服务
│   │   └── utils/         # 工具函数
│   ├── backend/           # 后端代码
│   │   ├── api/           # API 路由
│   │   ├── services/      # 业务逻辑
│   │   └── maa/           # MAA CLI 集成模块
│   └── shared/            # 前后端共享代码
├── public/                # 静态资源
├── config/                # 配置文件
├── .kiro/                 # Kiro 配置和指导文档
└── package.json           # 项目依赖
```

## 代码组织原则
- **模块化**: 按功能模块划分代码
- **MAA 集成**: 将 MAA CLI 调用封装在独立的服务模块中
- **前后端分离**: 清晰的前后端代码边界
- **配置管理**: MAA 配置文件的读写操作统一管理

## 核心模块
- **MAA 控制模块**: 启动、停止、状态查询
- **日志模块**: 实时日志查看和历史日志
- **配置管理模块**: MAA 配置的 CRUD 操作
- **资源管理模块**: MAA 资源文件的管理（待实现）
- **任务管理模块**: 任务调度和执行（待实现）

## 配置文件
- MAA CLI 配置文件路径需要在代码中配置
- WebUI 服务器配置（端口、日志等）
- 用户偏好设置
